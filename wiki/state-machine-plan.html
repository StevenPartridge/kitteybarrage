State Inventory

SitState Working

▶ Sit_Direction (forward = sit down, reverse = stand up pose)

  • Spawn pose: SitState.new(true, true) — holds last frame, paused
  • Normal entry: SitState.new() — plays full sit-down animation
  • Holds after animation completes. No auto-transition.
  • Exit cancels animation gate

StateName.SIT

StandUpState Working

◀ Sit_Direction reversed

  • Transitional — always chains to _next_state
  • Eases forward toward navigation_target during animation
  • Constructed: StandUpState.new(WalkState.new())

StateName.STANDUP

WalkState Working

↻ Walk_Direction (loop)

  • Moves toward navigation_target at speed
  • Updates facing_direction each frame
  • → SitState.new() on arrival (distance < 5px)
  • Pauses animation when no target

StateName.WALK

RunState Untested

↻ Run_Direction (loop)

  • Like WalkState × personality.run_speed_multiplier (1.6×)
  • → SitState.new() on arrival
  • Player: hold Shift while moving
  • AI: raise preference_run above 0

StateName.RUN

SprintState Untested

↻ Sprint_Direction (loop)

  • Like WalkState × personality.sprint_speed_multiplier (2.4×)
  • → SitState.new() on arrival
  • AI only by default; raise preference_sprint above 0

StateName.SPRINT

LayDownState Untested

▶ Lay_Direction (forward = lie down)

  • Transitional — chains to LayState
  • Plays Lay animation, waits for animation_finished
  • ⚠ Currently unreachable: preference_lay = 0.0 in default personality

StateName.LAY_DOWN

LayState Untested

⏸ Lay_Direction (last frame, paused)

  • Holds lying pose. No auto-transition.
  • Escaped only by begin_walk/run/sprint triggering SitUpState chain
  • Mirrors SitState's held-pose pattern

StateName.LAY

SitUpState Untested

◀ Lay_Direction reversed

  • Transitional — chains to _next_state
  • Plays Lay animation backwards (sitting up)
  • Uses long-awaited SITUP enum entry

StateName.SITUP

LookAroundState Untested

▶ LookAround_Direction (one-shot)

  • → SitState.new(true, true) when done
  • AI: preference_look_around = 0.3 — should trigger automatically
  • ⚠ No guard: can be called from non-SIT states

StateName.LOOK_AROUND

IdleState Deleted

Removed — never instantiated, no animation, no name() override

✓ FIXED — apply_decision(SIT) from LAY skips SitUp animation
Now chains SitUpState.new(SitState.new(true, true)) when current state is LAY or LAY_DOWN.
✓ FIXED — apply_decision(LAY) when already in LAY re-plays LayDownState
Now a no-op when current == LAY or LAY_DOWN.
