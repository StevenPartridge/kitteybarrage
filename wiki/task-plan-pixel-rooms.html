Pixel Rooms — Indoor World

Decided May 2026

ToffeCraft Pixel Rooms pack — asset survey complete. TileMap world, furniture scenes, hotspot slots, floor-type behavior, and a house layout for kitties to explore.

Goal & Scope

Give the kitties a real indoor world: rooms built from a pixel-art tileset, furnished with pieces they can navigate around, sit on, sniff, and knock off surfaces. Three interlocking systems:

  • TileMap world — floors and walls painted in Godot's TileMap GUI. Custom data layers on floor tiles encode floor type (stone, wood) which influences kitty behavior.
  • Furniture scenes — all furniture placed as individual scenes with collision, navigation avoidance, and named hotspot slots with finite capacity (sit ×N, lay ×N, sniff, knock).
  • Floor-type behavior — a two-tier lookup: first checks if the kitty is inside a rug/carpet's Area2D (→ RUG), then falls back to the TileMap tile type (STONE or WOOD). ActivityBrain applies bias modifiers at decision time.

Deliverable: a complete multi-room house kitties can explore autonomously. Outdoor assets and the Halloween pack are explicitly future scope.

Technical Context

Two-tier floor type lookup (new insight from survey)

The original plan assumed floor type came entirely from TileMap tile data. The survey revealed that rugs/carpets are standalone sprites — they are not tileset tiles. This requires a two-tier lookup:

# In Character._update_floor_type():
# Tier 1 — check if inside a rug Area2D (registered set)
for rug in _overlapping_rugs:
    if rug.get_overlapping_bodies().has(self):
        current_floor_type = FloorType.RUG
        return

# Tier 2 — read tile custom data from floor layer
var tile_data := floor_layer.get_cell_tile_data(
    floor_layer.local_to_map(floor_layer.to_local(global_position))
)
if tile_data:
    current_floor_type = FloorType[tile_data.get_custom_data("floor_type")]

The simplest implementation: Carpet scenes join a group "rug"; Character holds an Area2D that tracks overlapping rugs via area_entered/area_exited. No need for a registry — just a local set of overlapping rug areas on each Character.

Furniture variant selection

Each furniture file contains color variants in a grid. Selecting a variant at runtime uses Sprite2D.region_rect — set the atlas rect to the correct grid cell offset. This is the same pattern used for color variants on the kitty sprites.

# Example: Bed.gd — variant 0..3
@export var variant: int = 0

func _ready() -> void:
    var col := variant % 2
    var row := variant / 2
    $Sprite2D.region_rect = Rect2(col * 64, row * 64, 64, 64)

Navigation: what actually blocks

From the survey: sofas, beds, desks, drawers, bookcases, kitchen counters, oven, fridge, bathtub, toilet, and sink all block navigation. Plants, lamps, and small props do not. Doors in the wall layer must be marked walkable in the TileSet so the navmesh passes through them.

Decisions — Locked In

All five design questions answered. These are the architectural constraints implementation must follow.

1
How many TileMapLayer nodes?
A — Two layers: Floor + Walls

Floor layer carries all walkable tiles with floor_type custom data. Wall layer holds wall tiles, WalllMaker connectors, and door tiles. Wall decorations (windows, paintings, clocks) placed as Sprite2D scenes — gives per-item control and makes attaching windowsill hotspots trivial.

2
How do furniture color variants get selected?
A — Hand-pick per instance in the inspector

Each Furniture scene exports variant: int. Set it in the editor when placing furniture in the room. The room is a designed space — picking the teal sofa for the living room is a creative decision, not a code decision.

3
How does hotspot discovery work?
C + raycast filter + Door markers — proximity with line-of-sight

Core principle: kitties only act on furniture they can actually reach without going through a wall. Three parts working together:

1. Interest zone (Area2D, r≈200px): Character detects all nearby Furniture nodes reactively via signals. Area2D naturally goes through walls — this is intentional. A large zone means a kitty can "sense" the bed in the next room.

2. Raycast filter: Before claiming any hotspot, cast a ray from the character to the hotspot's world position against the walls physics layer only. If the ray hits — wall is in the way — skip that candidate. If the ray is clear, the hotspot is reachable. No room shapes, no topology graph — works automatically as you paint.

3. Door markers (EXPLORE hotspot): A small Door.tscn scene is placed at the center of each doorway gap. It exposes a single EXPLORE-type hotspot slot. The doorway is an open gap (no wall tile), so the raycast to a door marker always passes. When a kitty chooses EXPLORE, it navigates to the door — crossing into the adjacent room — where it then discovers new furniture via its interest zone. Curiosity weight in KittyPersonality drives how often EXPLORE is chosen vs. settling nearby.

Future hook: Food bowls, water dishes, and litter boxes are need-based global overrides — when hunger/thirst/urgency exceeds a threshold, the character bypasses the proximity system entirely and navigates to the nearest global target. This is a separate layer; the proximity + raycast architecture is unaffected.

4
Where does floor-type behavior live?
A — Cached on Character, polled every 0.5s

Character holds current_floor_type updated by a timer. Rug overlap is maintained reactively via area_entered/exited signals — the 0.5s timer only handles the TileMap tile query. ActivityBrain reads the cached value at decision time. No TileMap coupling in Brain or States.

5
How does furniture block navigation?
B — TileMap physics bake (walls) + NavigationObstacle2D (furniture)

Walls bake into the NavigationRegion2D via TileSet physics layer — hard navmesh blocks. Furniture scenes each carry a NavigationObstacle2D for dynamic soft avoidance. Agents route around walls at the macro level, then steer around furniture in real time. Idiomatic Godot 4 approach; sets up correctly for outdoor assets later.

Phase 5 — Hotspot System Complete Proximity + Raycast + Doors

No global registry. Each Character carries an interest zone (detects nearby furniture reactively) and a raycast filter (confirms line of sight before claiming). Door markers expose an EXPLORE hotspot that lets kitties deliberately cross into adjacent rooms.

FurnitureHotspot resource — slot ownership

# FurnitureHotspot.gd
class_name FurnitureHotspot
extends Resource

enum ActionType { SIT, LAY, SNIFF, KNOCK, EXPLORE }

@export var action: ActionType
@export var slots: Array[Vector2]  # world offsets from furniture origin
var _claimed: Dictionary = {}      # slot_index → character node
var knocked := false               # one-shot: permanently unavailable after knock

func claim(character) -> int:
    if knocked: return -1
    for i in slots.size():
        if not _claimed.has(i):
            _claimed[i] = character; return i
    return -1

func release(character) -> void:
    for key in _claimed.keys():
        if _claimed[key] == character:
            _claimed.erase(key); return

func has_available_slot() -> bool:
    return not knocked and _claimed.size() < slots.size()

Character — interest zone, raycast filter, slot ownership

# In Character.gd
@onready var _interest_zone: Area2D = $InterestZone  # CircleShape2D, r≈200px
var _nearby_furniture: Array[Furniture] = []
var _claimed_hotspot: FurnitureHotspot = null
const WALL_LAYER := 1  # physics layer bit for wall tiles — match your TileSet config

func _ready() -> void:
    _interest_zone.area_entered.connect(_on_furniture_entered)
    _interest_zone.area_exited.connect(_on_furniture_exited)

func _on_furniture_entered(area: Area2D) -> void:
    var f := area.get_parent() as Furniture
    if f and not _nearby_furniture.has(f): _nearby_furniture.append(f)

func _on_furniture_exited(area: Area2D) -> void:
    var f := area.get_parent() as Furniture
    if f: _nearby_furniture.erase(f)

# Returns true if no wall tile lies on the straight line to target_pos.
func _has_line_of_sight(target_pos: Vector2) -> bool:
    var space := get_world_2d().direct_space_state
    var query := PhysicsRayQueryParameters2D.create(
        global_position, target_pos, WALL_LAYER
    )
    query.exclude = [self]
    return space.intersect_ray(query).is_empty()

func find_nearby_hotspot(action: FurnitureHotspot.ActionType) -> Dictionary:
    for furniture in _nearby_furniture:
        for hs in furniture.get_hotspots():
            if hs.action != action or not hs.has_available_slot(): continue
            var slot_world := furniture.global_position + hs.slots[0]
            if not _has_line_of_sight(slot_world): continue  # wall in the way
            var idx := hs.claim(self)
            if idx >= 0:
                _claimed_hotspot = hs
                return { "world_pos": furniture.global_position + hs.slots[idx] }
    return {}

func release_hotspot() -> void:
    if _claimed_hotspot:
        _claimed_hotspot.release(self)
        _claimed_hotspot = null

Door marker scene — the EXPLORE hotspot

Door.tscn is a lightweight invisible marker placed at the center of each doorway gap in the editor. It extends Furniture, has no sprite and no collision, and returns a single EXPLORE hotspot at its own origin.

# Door.gd
class_name Door
extends Furniture

func get_hotspots() -> Array[FurnitureHotspot]:
    var hs := FurnitureHotspot.new()
    hs.action = FurnitureHotspot.ActionType.EXPLORE
    hs.slots = [Vector2.ZERO]  # slot IS the door position
    return [hs]
  • Scene tree: Door (Furniture) → DetectionArea (Area2D) + small CircleShape2D — just enough to enter kitty interest zones
  • No Sprite2D, no StaticBody2D, no NavigationObstacle2D — it's invisible and completely passthrough
  • The raycast from a kitty in the same room to the door's world position passes through the open gap — no wall tile there
  • The raycast from a kitty in the opposite room hits the wall — that door is not visible until the kitty approaches the gap from their side
  • Place one Door.tscn instance at each doorway when painting the house in Phase 7
EXPLORE slots don't stay claimed. Once the kitty arrives at the door position, it's crossed into the adjacent room and ActivityBrain fires a new decision cycle. The EXPLORE slot should be released immediately on arrival — no holding. The door is a waypoint, not a seat.

ActivityBrain — EXPLORE activity

  • Add EXPLORE to the set of possible activities in ActivityBrain._pick_activity()
  • EXPLORE weight comes from KittyPersonality.curiosity — a new personality field (float 0–1). High curiosity kitties wander frequently; homebodies rarely leave their favourite rug.
  • When EXPLORE is chosen: call character.find_nearby_hotspot(EXPLORE)
  • If a door is in range and passes the raycast → navigate to it; on arrival release the slot and re-decide
  • If no door in range → fall back to a random floor position anywhere in the navmesh (existing walk behaviour — kitties can still wander naturally)
  • All other activities (SIT, LAY, SNIFF) already query through find_nearby_hotspot() which applies the raycast filter — no further changes needed

State exit — slot release

  • SitState, LayState, future SniffState: call character.release_hotspot() in _exit()
  • MovementState interrupted mid-approach: also call release_hotspot() — the slot was pre-claimed at decision time
  • EXPLORE arrivals: release immediately in the movement arrival callback, not in a state's _exit()
  • KNOCK: KnockState sets hotspot.knocked = true on animation completion. Slot permanently gone — no release needed.
Phase 6 — Floor Type Behavior Complete

Two-tier floor query on Character, behavior bias modifiers in ActivityBrain.

# In Character.gd
enum FloorType { NONE, WOOD, STONE, RUG }

var current_floor_type: FloorType = FloorType.NONE
var _overlapping_rugs: Array = []  # maintained via area_entered/exited on _floor_sensor

func _update_floor_type() -> void:
    # Tier 1: rug presence (reactive, maintained by signals)
    if not _overlapping_rugs.is_empty():
        current_floor_type = FloorType.RUG
        return
    # Tier 2: tile data
    var cell := floor_layer.local_to_map(floor_layer.to_local(global_position))
    var td := floor_layer.get_cell_tile_data(cell)
    if td:
        var raw: String = td.get_custom_data("floor_type")
        current_floor_type = FloorType[raw] if FloorType.has(raw) else FloorType.NONE
    else:
        current_floor_type = FloorType.NONE

  • Kitty spawns and navigates within the room without clipping through any wall tile
  • Kitty routes around a placed sofa — does not walk through it
  • Kitty in the living room cannot claim the bedroom bed — raycast hits the wall between them
  • Kitty standing in a doorway gap can claim furniture on both sides — raycast is clear in both directions
  • Kitty chooses EXPLORE, navigates to the Door marker, crosses into the adjacent room, then re-decides and finds new furniture
  • High-curiosity kitty visibly wanders between rooms more often than a low-curiosity kitty over a 2-minute observation window
  • Kitty navigates through a doorway into an adjacent room without getting stuck on door frame corners
  • Multiple kitties navigate simultaneously — no deadlock, no stacking
  • Kitty claims a sit hotspot on the bed, navigates to the slot position, and enters SitState
  • Two kitties claim two different slots on the same sofa simultaneously
  • Third kitty denied the sofa (all slots taken) — falls back to floor sitting
