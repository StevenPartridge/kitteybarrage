Blocking Furniture

These pieces need full collision + nav obstacle so kitties path around them. Each one goes in scenes/furniture/.

Node tree — blocking furniture
  • Node2D script: Furniture.gd definition: resources/furniture/[type].tres
  • └ Sprite2D texture only — region set by code
  • └ StaticBody2D collision layer 1
  • └ CollisionShape2D
  • └ NavigationObstacle2D avoidance enabled
  • └ DetectionArea (Area2D) group: furniture_area
  • └ CollisionShape2D circle ~48px

No subclass script — every piece uses Furniture.gd. Hotspot data and sprite region come from the definition resource set in the inspector. The variant property (default 0) selects which spritesheet cell to show.

Decorations

No collision, no nav obstacle. Kitties walk through them visually but can sniff if they're close. Goes in scenes/decorations/.

Node tree — decoration (non-blocking)
  • Node2D script: Furniture.gd definition: resources/furniture/[type].tres
  • └ Sprite2D texture only — region set by code
  • └ DetectionArea (Area2D) group: furniture_area
  • └ CollisionShape2D circle ~32px

No StaticBody2D. No NavigationObstacle2D. Same Furniture.gd script — the missing physics nodes are simply absent.

Window exception: Window.tscn has a windowsill sit + lay hotspot — still non-blocking (no StaticBody2D) but does need a DetectionArea. Use Furniture.gd with resources/furniture/window.tres as the definition. Place it flush against the wall tile in the editor.

Markers

Invisible scenes placed in the editor at the center of each doorway gap. No sprite, no collision — just a small DetectionArea so kitties can sense them from the adjacent room. Goes in scenes/markers/.

Node tree — door marker
  • Door (extends Furniture)
  • └ DetectionArea (Area2D) group: furniture_area
  • └ CollisionShape2D circle ~24px

No Sprite2D. No StaticBody2D. No NavigationObstacle2D. Completely invisible and passthrough at runtime.
