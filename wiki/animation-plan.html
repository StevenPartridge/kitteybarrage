Naming convention — critical. Every animation name in SpriteFrames must match exactly what the code constructs via Global.get_full_animation_name(state_name, direction), which produces "StateName_DirectionName". Example: play_loop("Walk", SOUTH) looks for "Walk_South". Case sensitive. If the names don't match, AnimationController pushes an error and nothing plays.

Direction suffix reference

_North   _Northeast   _East   _Southeast

_South   _Southwest   _West   _Northwest

Each animation needs all 8 variants. So Lay = 8 entries: Lay_North, Lay_Northeast, … Lay_Northwest.

Phase 1 — Art Setup (Godot Editor)

All Phase 1 work happens in the SpriteFrames resource attached to the AnimatedSprite2D node on the Kitty scene. No code changes in Phase 1 — just importing and naming animations. Phase 2 code will fail gracefully (push_error, no crash) until the names are in place.
  1. Import and name the LookAround animations — 8 entries: LookAround_North, LookAround_Northeast, … LookAround_Northwest. 5 frames each. One-shot (not looping).
  2. Import and name the Lay animations — 8 entries: Lay_North … Lay_Northwest. 8 frames each. One-shot forward (lying down); reverse (sitting up) is handled in code — no separate asset.
  3. Import and name the Run animations — 8 entries: Run_North … Run_Northwest. 5 frames each. Looping.
  4. Import and name the Sprint animations — 8 entries: Sprint_North … Sprint_Northwest. 8 frames each. Looping.

Phase 2 — Code Implementation

States being added

  • LookAroundState — plays LookAround, returns to SitState
  • LayDownState — plays Lay forward, chains to LayState
  • LayState — holds lying pose (Lay last frame, paused)
  • SitUpState — plays Lay in reverse, chains to SitState (uses existing SITUP enum)
  • RunState — like WalkState but plays Run loop
  • SprintState — like RunState but plays Sprint animation

Existing code being updated

  • Global.StateName — add LOOK_AROUND, SPRINT
  • KittyPersonality — add preference_look_around, duration_look_around, preference_sprint, duration_sprint
  • ActivityBrain._init — update required-keys validation list
  • Kitty.apply_decision — handle LAY → LayDownState, LOOK_AROUND, RUN, SPRINT
  • Kitty.begin_run() — new method for player-triggered running
  • KittyDirector — run key detection

Phase 2a · Laying down

Note: LayDownState needs its own enum entry LAY_DOWN (distinct from LAY used by LayState). Both states returning LAY from get_state_name() would cause the FSM to reject same-name transitions silently.
  1. Create LayDownState.gd — Constructor: _init(next_state: State). Plays Lay one-shot forward. On animation_finished, returns next_state. Follows StandUpState pattern exactly. Uses LAY_DOWN enum entry. scripts/kitty_states/LayDownState.gd (new)
  2. Create LayState.gd — Plays Lay paused at the last frame (lying pose). Holds until externally transitioned. Mirrors how SitState holds the sitting pose. Uses LAY enum entry. scripts/kitty_states/LayState.gd (new)
  3. Create SitUpState.gd — Plays Lay in reverse (uses play_once("Lay", direction, reverse: true)). Uses existing SITUP entry in Global.StateName. scripts/kitty_states/SitUpState.gd (new)
  4. Wire LAY flow into Kitty.apply_decision — LAY decision → LayDownState.new(LayState.new()). Split from the current SIT, LAY → SitState.new() branch. Handle getting up: when a WALK decision arrives while in LAY, chain SitUpState → SitState → StandUpState → WalkState. scripts/Kitty.gd:64

Phase 2b · Look Around

  1. Create LookAroundState.gd — Plays LookAround one-shot. On finish → SitState.new(true, true) (back to sitting pose, paused). Uses LOOK_AROUND in Global.StateName. scripts/kitty_states/LookAroundState.gd (new)
  2. Wire LOOK_AROUND as an AI activity — Add LOOK_AROUND to Global.StateName. Add preference_look_around and duration_look_around to KittyPersonality. Update ActivityBrain._init required-keys list. Handle LOOK_AROUND decision in Kitty.apply_decision → LookAroundState.new(). scripts/Global.gd, scripts/KittyPersonality.gd, scripts/ActivityBrain.gd:11, scripts/Kitty.gd

Phase 2c · Running (Run + Sprint, AI and player)

  1. Add SPRINT to Global.StateName and KittyPersonality — New SPRINT enum entry. Add preference_sprint and duration_sprint fields. Update ActivityBrain._init required-keys list. scripts/Global.gd:5, scripts/KittyPersonality.gd, scripts/ActivityBrain.gd:11
  2. Create RunState.gd and SprintState.gd — Both follow WalkState closely. RunState plays Run; SprintState plays Sprint. Speed multiplier from KittyPersonality. scripts/kitty_states/RunState.gd (new), scripts/kitty_states/SprintState.gd (new)
  3. Wire run trigger — both AI and player — Handle RUN and SPRINT decisions in Kitty.apply_decision. Add Kitty.begin_run(). Detect run key in KittyDirector._physics_process. scripts/Kitty.gd, scripts/KittyDirector.gd
