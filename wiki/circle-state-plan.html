CircleState — Implementation Plan

Done

Kitty Barrage · Updated May 2026 · Verified against current codebase

Decisions Locked In

1
Animation approach: B — Dedicated Circle animation per direction

One full 360° clockwise animation called "Circle". No directional suffix — direction changes mid-animation. Flip horizontally for counterclockwise. Sub-frame playback (start_progress) handles partial arcs like a 90° pivot.

Code impact: AnimationController needs two additions: set_flip_h(enabled: bool) and play_once_raw(anim_name, start_progress) that bypass the Global.get_full_animation_name suffix convention. CircleState calls these directly rather than using play_loop / play_once.
2
Direction of rotation: B — Match shortest arc to new facing direction

For the pre-lay circle, some cats will only go one direction — this is personality data. For pivots (Phase 2), shortest arc wins. These have different enough logic that they warrant two states: PreLayCircleState and TurnCircleState (Phase 2), both using the same "Circle" animation asset.

Code impact: PreLayCircleState reads entity.personality.circle_clockwise_only: bool to decide direction. TurnCircleState (Phase 2) computes shortest arc from entity.facing_direction to new target heading.
3
Direction-change pivot trigger: C — Phase 2, skip for now

Build PreLayCircleState and stand-to-walk circling first. TurnCircleState and WalkState angle-threshold detection come after the core is working and tuned.

4
Circle parameters: A — Inspector-editable via KittyPersonality

Adds a new "Circling" export group to KittyPersonality with radius, rotations, speed multiplier, and per-direction preference flags.

Code impact: KittyPersonality.gd gets new fields. PreLayCircleState._enter_state reads them from entity.personality.

How CircleState Fits the State Machine

Phase 1 flows

WALK → LAY decision → PRE_LAY_CIRCLE → LAY

SIT → STANDUP → PRE_LAY_CIRCLE → WALK

Stand-to-walk circling is optional — gated by personality.circle_before_walk: bool. Off by default, on for extra-fussy kitties.

Phase 2 flows (later)

WALK → (sharp turn) → TURN_CIRCLE → WALK

TurnCircleState carries the pending NavigationTarget through the turn — exits into a new WalkState already aimed at the right target.

Both states follow the StandUpState pattern — constructor takes next_state: State, sets _done = true when the arc completes, returns _next_state from tick(). The stretch goal of a TransitionalState base class would consolidate this pattern across StandUpState, PreLayCircleState, and TurnCircleState.

Animation Asset Specification

PropertyValue
Name in SpriteFrames"Circle" exactly — no directional suffix. AnimationController uses play_once_raw("Circle", ...) which skips Global.get_full_animation_name.
Direction of travelClockwise. Counterclockwise is a free flip_h of the same frames.
Starting facingSouth — matches Global.Direction.SOUTH and Kitty's spawn default.
Full rotation360° of arc = one full loop of the animation. Sub-arcs (90°, 180°) are played using start_progress and end_progress in code — no separate assets needed.
Recommended framesMinimum 8 frames per 90° of arc (32 frames for a full 360°) for smooth animation at typical game speeds.
Radius baked in?No — the sprite shows the cat walking in its typical stride. The radius of the on-screen circle is controlled by personality.circle_radius in code.
What changes per frameThe cat's facing direction rotates clockwise. At frame 0: facing South. At ~25%: facing West. At ~50%: facing North. At ~75%: facing East. At 100%: back to South.
Flip behaviourWhen set_flip_h(true), the sprite mirrors horizontally — the cat appears to walk counterclockwise.
Don't use the standard Walk_Direction naming convention for this animation. The existing play_loop("Walk", direction) and play_once("Walk", direction) on AnimationController append a directional suffix. The Circle animation has no suffix — CircleState uses the new play_once_raw and play_loop_raw methods added in Phase 1.

scripts/AnimationController.gd

Add three new methods. Existing interface (play_loop, play_once, cancel, pause, get_playback_progress, animation_finished) is unchanged.

func set_flip_h(enabled: bool) -> void:
    _player.flip_h = enabled

func play_loop_raw(anim_name: String) -> void:
    if _is_loop and _current_state_name == anim_name:
        return
    _cancel_silent()
    if not _player.sprite_frames.has_animation(anim_name):
        push_error("Animation not found: " + anim_name)
        return
    _current_state_name = anim_name
    _is_loop = true
    _is_reverse = false
    _player.play(anim_name)

func play_once_raw(anim_name: String, start_progress: float = 0.0) -> void:
    _cancel_silent()
    if not _player.sprite_frames.has_animation(anim_name):
        push_error("Animation not found: " + anim_name)
        return
    _current_state_name = anim_name
    _is_loop = false
    _is_reverse = false
    _player.animation_finished.connect(_on_animation_finished)
    _player.play(anim_name)
    if start_progress > 0.0:
        var total: int = _player.sprite_frames.get_frame_count(anim_name)
        _player.frame = clamp(int(start_progress * float(total)), 0, total - 1)

scripts/kitty_states/PreLayCircleState.gd (new)

Follows the StandUpState pattern: constructor takes next_state, tick() returns it when done. Reads circle parameters from entity.personality.

extends State
class_name PreLayCircleState

var _next_state: State
var _angle_covered: float = 0.0
var _clockwise: bool = true
var _target_angle: float = 0.0

func _init(next_state: State) -> void:
    _next_state = next_state

func name() -> Global.StateName:
    return Global.StateName.CIRCLE

func _enter_state(_from: Global.StateName) -> void:
    assert(entity != null, "PreLayCircleState requires a Kitty entity")
    _angle_covered = 0.0
    _target_angle = TAU * entity.personality.circle_rotations
    if entity.personality.circle_clockwise_only:
        _clockwise = true
    else:
        _clockwise = entity.facing_direction in [
            Global.Direction.NORTH, Global.Direction.NORTHEAST,
            Global.Direction.EAST,  Global.Direction.SOUTHEAST
        ]
    entity.anim.set_flip_h(not _clockwise)
    entity.anim.play_loop_raw("Circle")

func tick(delta: float) -> State:
    var angular_speed: float = (entity.speed * entity.personality.circle_speed_multiplier) \
                               / entity.personality.circle_radius
    _angle_covered += angular_speed * delta
    var orbit_angle: float = _angle_covered * (1.0 if _clockwise else -1.0)
    var offset := Vector2(
        cos(orbit_angle) * entity.personality.circle_radius,
        sin(orbit_angle) * entity.personality.circle_radius
    )
    entity.velocity = offset.normalized() * entity.speed * entity.personality.circle_speed_multiplier
    entity.move_and_slide()
    if _angle_covered >= _target_angle:
        return _next_state
    return null

func _exit_state() -> void:
    entity.anim.set_flip_h(false)
    entity.anim.cancel()
    entity.velocity = Vector2.ZERO

Phase 2 Preview — TurnCircleState

When implemented, TurnCircleState will:

  • Be created by WalkState.tick() when the angle delta between entity.facing_direction and new target heading exceeds personality.pivot_angle_threshold (suggested default: 120°)
  • Accept the pending NavigationTarget in its constructor so the kitty exits already aimed at the right target
  • Use shortest-arc rotation — always. No personality override.
  • Play a partial circle arc, not a full rotation — start_progress on play_once_raw will handle this
  • Transition back to WalkState.new() with the carried target

New KittyPersonality field needed: @export var pivot_angle_threshold: float = 120.0

Stretch Ideas

Emit circle_completed on Kitty — fired when PreLayCircleState._exit_state runs. Future toys and interactive objects subscribe without touching the state machine. One signal, non-breaking.
TransitionalState base class — StandUpState, PreLayCircleState, and (Phase 2) TurnCircleState all share _next_state: State, _done: bool, and the tick-returns-next pattern. A shared base makes the pattern explicit and halves the boilerplate for future transitional states.
Trajectory abstraction — CircleState is the first state with a continuous spatial path rather than "move toward a point." A Trajectory seam sampled each tick opens the door to FigureEightState, WanderState, etc.
